# MCP Token Cost Subproject

This subproject tracks token consumption before and after migrating MCP tool flows to a lower-token architecture.

## Goal

Reduce token usage per turn while preserving response quality.

## What Is Measured

Per turn baseline includes:

- estimated input tokens
- estimated output tokens
- estimated total tokens
- attempts per turn
- continuation retry rate
- overflow retry rate
- compact/hard-compact rates

## Data Source

The parser reads JSONL logs generated by the provider.

Preferred fields:

- `chat.turn.complete.data.metrics.estimatedOutputTokens`
- `chat.turn.complete.data.contextUsage.*`

Legacy fallback (older log format):

- `prompt_n` and `predicted_n` parsed from `chat.stream.chunk.data.text`

## Commands

Create baseline:

```bash
node subprojects/mcp-token-cost/token-baseline.mjs \
  --input logs.log \
  --output subprojects/mcp-token-cost/baseline/baseline-before-mcp-migration.json
```

Create model-specific baseline (example for Qwen):

```bash
node subprojects/mcp-token-cost/token-baseline.mjs \
  --input "<LATEST_JSONL_LOG_PATH>" \
  --model-contains "qwen" \
  --output subprojects/mcp-token-cost/baseline/qwen36-before-api-direct.json
```

Compare before/after:

```bash
node subprojects/mcp-token-cost/compare-baseline.mjs \
  --before subprojects/mcp-token-cost/baseline/baseline-before-mcp-migration.json \
  --after subprojects/mcp-token-cost/baseline/baseline-after-mcp-migration.json
```

Build tool inventory (full + used candidates):

```bash
node subprojects/mcp-token-cost/tool-inventory.mjs \
  --input logs.log \
  --output-json subprojects/mcp-token-cost/baseline/tool-inventory.json \
  --output-md subprojects/mcp-token-cost/baseline/tool-inventory.md
```

## Evaluation Rule

Migration is considered successful when:

1. Average total tokens/turn decreases.
2. P95 total tokens/turn decreases.
3. Retry rates do not worsen significantly.
4. Functional behavior remains unchanged.
