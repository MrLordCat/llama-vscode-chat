# Инструменты для AI-агентов: Экономия токенов и эффективность

> Этот документ содержит рекомендации по CLI-инструментам, которые экономят токены
> при работе AI-агентов (Copilot, Cursor, Claude Code и т.д.) в терминале.

## Почему это важно

AI-агенты в VS Code используют терминал для выполнения команд. Каждая команда
потребляет токены:
- **Входные токены**: сама команда + контекст
- **Выходные токены**: результат команды, который агент читает и обрабатывает

**Ключевой принцип**: чем точнее и компактнее результат команды, тем меньше токенов
тратится на обработку. Инструменты ниже оптимизированы именно для этого.

---

## 🏆 Tier 1: Критически важные (максимальная экономия)

### 1. ripgrep (`rg`)
**Что это**: Самый быстрый инструмент поиска по тексту (Rust).
**Почему экономит токены**:
- Возвращает только точные совпадения с контекстом (не весь файл)
- Игнорирует `.gitignore` по умолчанию (меньше шума)
- Флаги `-l` (только имена файлов), `-c` (счётчик), `-n` (номера строк) дают компактный вывод
- Замена `grep -r` с 10x+ производительностью

**Типичное использование агентом**:
```bash
# Поиск по коду — возвращает только нужные строки
rg "function.*main" src/

# Только имена файлов (минимальный вывод)
rg -l "TODO" src/

# Подсчёт совпадений (1 число вместо списка)
rg -c "console\.log" src/

# Поиск с ограничением контекста
rg -C 2 "error" src/  # 2 строки контекста
```

**Установка**: `winget install BurntSushi.ripgrep`

---

### 2. fd (`fd-find`)
**Что это**: Быстрый поиск файлов (Rust), замена `find`.
**Почему экономит токены**:
- Игнорирует `.gitignore` и скрытые файлы по умолчанию
- Выводит только релевантные файлы (нет системного мусора)
- Быстрая фильтрация по имени/маске

**Типичное использование агентом**:
```bash
# Найти все TypeScript файлы
fd "\.ts$" src/

# Найти конкретный файл
fd "package\.json"

# Глубина поиска ограничена (меньше токенов)
fd -d 3 "\.test\.ts$"
```

**Установка**: `winget install SharkDP.fd`

---

### 3. jq
**Что это**: Командная строка для обработки JSON.
**Почему экономит токены**:
- Извлекает только нужные поля из больших JSON (вместо чтения всего файла)
- Форматирует/сжимает JSON для компактного вывода
- Фильтрация и агрегация на стороне клиента

**Типичное использование агентом**:
```bash
# Только имена ключей
jq 'keys' package.json

# Только конкретное поле
jq '.version' package.json

# Список зависимостей (компактный)
jq '.dependencies | keys[]' package.json

# Сжатый вывод (меньше токенов)
jq -c '.scripts' package.json
```

**Установка**: `winget install jqlang.jq`

---

### 4. bat
**Что это**: Улучшенная `cat` с подсветкой синтаксиса и номерами строк.
**Почему экономит токены**:
- Флаги `-n` (номера строк) помогают агенту ссылаться на строки
- `--line-range` читает только нужный диапазон (не весь файл)
- Подсветка синтаксиса помогает быстрее понять структуру

**Типичное использование агентом**:
```bash
# Только конкретный диапазон строк (экономит токены!)
bat --line-range 1:50 src/main.ts

# Без заголовков (чистый вывод)
bat --style=numbers --color=never file.ts

# С подсветкой (помогает понять структуру)
bat --color=always file.ts
```

**Установка**: `winget install sharkdp.bat`

## 🥈 Tier 2: Высокая ценность

### 5. fzf
**Что это**: Командный фаззер для интерактивного поиска.
**Почему экономит токены**:
- Фильтрация результатов других команд в реальном времени
- Позволяет агенту быстро сузить выборку

**Типичное использование агентом**:
```bash
# Комбинация с fd
fd -t f | fzf

# Комбинация с rg
rg -l "pattern" | fzf
```

**Установка**: `winget install junegunn.fzf`

---

### 6. delta
**Что это**: Улучшенный просмотр diff (git diff).
**Почему экономит токены**:
- Компактный и читаемый вывод изменений
- Синтаксис-подсветка в diff
- Фильтрация пустых изменений

**Типичное использование агентом**:
```bash
# Красивый diff
git diff | delta

# Только изменённые файлы
git diff --name-only
```

**Установка**: `winget install dandavison.delta`

---

### 7. tokei
**Что это**: Быстрый подсчёт строк кода (Rust).
**Почему экономит токены**:
- Мгновенный подсчёт LOC без чтения файлов
- JSON-вывод для программной обработки
- Помогает понять масштаб проекта за 1 команду

**Типичное использование агентом**:
```bash
# Общий подсчёт
tokei

# По языкам
tokei --output-json

# Конкретная директория
tokei src/
```

**Установка**: `winget install XAMPPRocky.tokei`

---

### 8. zoxide
**Что это**: Умная навигация по директориям.
**Почему экономит токены**:
- Быстрый переход в часто используемые директории
- Не нужно писать полные пути (меньше токенов в команде)

**Типичное использование агентом**:
```bash
# Переход в часто используемую директорию
zoxide query "src"
```

**Установка**: `winget install ajeetdsouza.zoxide`

---

## 🥉 Tier 3: Полезные дополнения

### 9. hyperfine
**Что это**: Бенчмарк команд в терминале.
**Почему экономит токены**:
- Быстрое сравнение производительности команд
- Помогает выбрать оптимальную команду

**Установка**: `winget install sharkdp.hyperfine`

---

### 10. sd
**Что это**: Быстрая замена текста в файлах (Rust), замена `sed`.
**Почему экономит токены**:
- Простой синтаксис (меньше ошибок)
- Рекурсивная замена по маске

**Установка**: `winget install burntsushi.sd`

---

### 11. watchexec / watch
**Что это**: Запуск команд при изменении файлов.
**Почему экономит токены**:
- Автоматический мониторинг (не нужно повторять команды)

**Установка**: `winget install watchexec`

---

### 12. tree
**Что это**: Древовидная структура директорий.
**Почему экономит токены**:
- Компактный обзор структуры проекта
- Флаги для ограничения глубины

**Установка**: `winget install tree` (или встроен в WSL)

## 🎯 Паттерны экономии токенов

### Паттерн 1: Поиск вместо чтения файлов
```bash
# ❌ ПЛОХО: Чтение всего файла (много токенов)
cat src/large-file.ts

# ✅ ХОРОШО: Поиск конкретной информации
rg "class.*Component" src/large-file.ts
```

### Паттерн 2: Фильтрация на стороне инструмента
```bash
# ❌ ПЛОХО: Чтение всего JSON
cat package.json

# ✅ ХОРОШО: Извлечение нужного поля
jq '.version, .name' package.json
```

### Паттерн 3: Ограничение вывода
```bash
# ❌ ПЛОХО: Весь список файлов
ls -R src/

# ✅ ХОРОРОШО: Только нужные файлы
fd "\.ts$" src/ | head -20
```

### Паттерн 4: Подсчёт вместо списка
```bash
# ❌ ПЛОХО: Список всех совпадений
rg "TODO" src/

# ✅ ХОРОШО: Только количество
rg -c "TODO" src/
```

### Паттерн 5: Диапазон строк вместо всего файла
```bash
# ❌ ПЛОХО: Весь файл
cat src/extension.ts

# ✅ ХОРОШО: Только нужный диапазон
bat --line-range 1:50 --style=numbers src/extension.ts
```

brew install ripgrep fd jq bat fzf delta tokei zoxide hyperfine sd

## 📊 Оценка экономии токенов

| Инструмент | Типичная экономия | Приоритет |
|---|---|---|
| `rg` | 60-80% vs `grep -r` | 🔴 Критический |
| `fd` | 50-70% vs `find` | 🔴 Критический |
| `jq` | 70-90% vs `cat JSON` | 🔴 Критический |
| `bat` | 40-60% vs `cat` | 🟡 Высокий |
| `fzf` | 30-50% (фильтрация) | 🟡 Высокий |
| `delta` | 30-40% (diff) | 🟡 Высокий |
| `tokei` | 80-90% vs чтение файлов | 🟢 Средний |
| `zoxide` | 20-30% (команды) | 🟢 Средний |

## 🔧 Конфигурация для агентов

### .gitignore для агентов
Добавьте в корень проекта:
```
# Игнорируется rg и fd автоматически
node_modules/
dist/
build/
*.log
```

### Alias для bash/zsh
```bash
# Быстрые алиасы
alias sr='rg'           # search in repo
alias ff='fd'           # find files
alias jj='jq .'         # pretty JSON
alias tt='tokei'        # count lines
alias dd='git diff | delta'  # pretty diff
```
